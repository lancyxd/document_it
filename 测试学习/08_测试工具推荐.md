#  1、压力测试工具推荐

```shell
AB （apache bench）是apache下的一个工具，主要用于对web站点做压力测试
Benchmark
JM(Jmeter)
LR(Loadrunner)    # 收费
平安Power
WB
备注： teamview远程控制工具 

```

- ab

  适用场景: 单个接口的测试。

  优点：简单和实用。 缺点:缺乏对整个业务流程的支持。

- jmeter

  适用场景: 简单的并发测试，整个业务流程测试，组合并发测试，csv动态导入变量，录制测试脚本。

  优点：功能全，可扩展性高，易学易操作。缺点：性能不稳定，高并发下通过界面实时观察结果集，容易造成界面卡死或者崩溃。

- loadrunner

  适用场景：基本包括了jmeter常用的功能，支持ip欺骗。

  优点: 更加专业、稳定和高效。

  缺点: 非开源免费，售价高。

## 1.1 ab工具

[性能测试工具AB](https://www.cnblogs.com/ailiailan/p/10239849.html)

- 请求命令   getuser.json即请求body体的内容

ab -c 1 -n 1 -p 'getuser.json' -T 'application/json' 'http://127.0.0.1:2600/v1/search/tenant/info/origin'

- 请求参数解析

  ```shell
  -n 总请求次数（最小默认为1）
  -c 并发次数（最小默认为1，且不能大于总请求数，例如：10个请求，10个并发，实际就是1人请求一次）
  -p POST参数文档路径（-p 和 -T 参数要配合使用）
  -T POST数据所使用的Content-type头信息（header 头内容类型）
  -s timeout    #指定每个请求的超时时间，默认是30秒
  -t 测试所进行的最大秒数。其内部隐含值是-n 50000，它可以使对服务器的测试限制在一个固定的总时间以内。默认时，没有时间限制。
  
  -v verbosity    #设置详细模式等级
  -w    #将结果输出到html的表中
  -i    #使用HEAD方式代替GET发起请求
  -k 启用HTTP KeepAlive功能，即在一个HTTP会话中执行多个请求，默认时，不启用KeepAlive功能
  
  ab -n 100  -c 10 -p 'post.txt' -T 'application/x-www-form-urlencoded' 'http://test.api.com/test/register'  # 测试并发用户数为10，请求数量为100的post请求
  ```

- 返回结果分析

  Requests per second 吞吐率：指某个并发用户下单位时间内处理的请求数。

  Time per request上面的为用户平均请求等待时间，指处理完所有请求所花费的时间/(总请求数/并发用户数)

  Time per request下面的是服务器平均请求处理时间，指处理完成所有请求所花费的时间/总请求数

  Percentage of the requests served within a certain time：每秒请求时间分布情况，指在整个请求中，每个请求的时间长度的分布情况，例如有 50% 的请求响应在 8ms 内，66% 的请求响应在 10ms 内，说明有 16% 的请求在 8ms~10ms 之间。
